# AquaDesk — Паспорт проекта

## Что это

Telegram Web App для управления фитнес-центром с бассейном.
Тренеры и координаторы ведут клиентов, тренировки, группы, дежурства и расчёт зарплаты прямо в Telegram.

## Стек

- **Frontend**: Vanilla JS (ES2020), без фреймворков. UI разбит на 6 модулей `frontend/js/app*.js` + слой Supabase `frontend/js/db.*.js` (8 доменных файлов: core/clients/groups/schedule/analytics/ops/salary/misc). Порядок загрузки в `index.html` критичен (см. ARCHITECTURE.md)
- **Backend**: Supabase (PostgreSQL + RLS + RPC)
- **Деплой**: GitHub Pages, скилл `aquadesk-deploy`
- **Вход**: Telegram Web App (`window.Telegram.WebApp.initDataUnsafe.user.id`)

## Ключевые бизнес-правила

### Тарифы (config.js)
```
Кат.1 = 85 000 сум/ПТ
Кат.2 = 110 000 сум/ПТ
Кат.3 = 130 000 сум/ПТ
Дежурство = 14 000 сум/час
Группы взрослые: до 3 чел=110к, до 6=120к, 7+=130к
Группы дети: 40% от оплат клиентов (group_children_pct)
PT_PRICES — клиентские цены ПТ для расчётной выручки CEO-аналитики
  (пока = тарифам тренеров, ⚠️ уточнить по реальному прайсу)
```

### Пакеты абонементов (SUB_PACKAGES)
```
Дети:     5 ПТ/2 нед | 10 ПТ/1 мес | 25 ПТ/3 мес | 50 ПТ/6 мес | Викенд 5 ПТ/1 мес (только сб/вс)
Взрослые: 5 ПТ/2 нед | 10 ПТ/1 мес | 25 ПТ/3 мес
```
> Пакет «Викенд» (детский): `is_weekend` на clients+subscriptions; списание ПТ в будни жёстко заблокировано (`isWeekendDate` в config.js).

### Разовые посещения
Три варианта: dropin1 (кат.1), dropin2 (кат.2), dropin3 (кат.3).
Дети могут использовать разовое только 1 раз.

### Лимиты дат
- Тренировку можно внести за последние **72 часа** (MAX_BACKDATE_HOURS), позже — запрос на одобрение
- Конспект — дедлайн **48 часов** после тренировки (NOTE_DEADLINE_HOURS)
- Редактирование/удаление записи — **30 минут** (EDIT_WINDOW_MIN), позже — запрос на одобрение

### Критичные правила работы с данными
- Профили сотрудников **не удалять** — только архив (CASCADE снесёт их клиентов, см. DATABASE.md)
- Pending-запросы (удаление клиента/ПТ) защищены от дублей на уровне db.js
- Критичные операции оборачивать в `once(key, fn)` / `_pending` Set
- Кеш: `cached(key, fn, ttl=300000)`, сбрасывать `invalidateCache('profiles')` после записи

1. Пушить тренеров к использованию — повысить вовлечённость, напоминания, онбординг
2. Интеграция с 1С — Step 1 (подтверждение списаний через панель «Ресепшн») реализован. Далее: Step 2/3 (двусторонняя сверка / прямой API 1С)
3. Клиентский опыт — детали не конкретизированы
4. Договор / ТЗ — формализация
5. Staging-среда — git ветка `staging` + Supabase Branch + Netlify для preview URL ($0.013/час, только пока активна)
6. Система онбординга — для новых тренеров и координаторов
7. Клиентская панель — отдельный интерфейс для клиентов фитнес-центра

### На рассмотрении
- Step 3 интеграции с 1С (прямой API) — нужно выяснить тип 1С и наличие разработчика
- Telegram-бот в группу + Google Sheet (вариант 2) — бот читает сообщения тренеров, парсит через Claude API (~$20–40/мес) и пишет в Google Sheet. Хостинг Railway/Render (free tier / от $5/мес), Sheets API и бот бесплатно. Риск: неоднозначные сообщения → бот может внести не то / не туда (чувствительно для детских групп)
